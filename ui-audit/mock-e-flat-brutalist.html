I need to refactor our authentication module. Currently it's a 500-line monolith with sessions, JWT, OAuth, and password hashing all mixed together. What's a good way to break this apart?

assistant [claude-4-sonnet] 15:22:11

Here's a solid approach to decompose a monolithic auth module:

1. Separate by concern:

auth/
├── __init__.py          # Public API re-exports
├── passwords.py         # Hashing, verification
├── tokens.py            # JWT creation, validation
├── sessions.py          # Session management
├── oauth.py             # OAuth provider integration
└── middleware.py         # Request authentication

2. Define clear interfaces: Each module should expose a small set of functions. For example, passwords.py only needs hash_password() and verify_password().

3. Use dependency injection: Don't hardcode which token backend or session store is used.
